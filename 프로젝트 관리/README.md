# EnglishAgentHub 프로젝트 목표

## 1. 프로젝트 한 줄 정의

EnglishAgentHub는 사용자가 직접 등록한 OpenAI API Key를 기반으로, 영어 스피킹 연습과 AI 피드백을 제공하는 인증형 학습 에이전트 플랫폼이다.

## 2. 핵심 목표

- OpenAI Realtime API를 활용해 실제 회화에 가까운 영어 스피킹 코칭 경험을 제공한다.
- 사용자별 API Key, 학습 이력, 대화 기록, 에이전트 설정을 안전하게 분리해 관리한다.
- 프롬프트 설계, 음성 대화, 피드백 생성, 학습 데이터 구조화를 하나의 서비스 흐름으로 연결한다.
- 기존 auth/admin/RBAC 기반을 유지하면서 AI 학습 서비스 도메인을 단계적으로 확장한다.

## 3. 채용 공고 요구사항 반영 방향

참고 채용 공고의 핵심은 생성형 AI 기반 문항 생성 파이프라인, LLM API 연동, 프롬프트 설계, 과목별 데이터 구조화, TypeScript 백엔드 및 REST API 경험이다. EnglishAgentHub에서는 이를 다음과 같이 반영한다.

| 공고 요구사항 | 프로젝트 반영 방향 |
| --- | --- |
| 생성형 AI 기반 콘텐츠 생성 파이프라인 | 영어 대화 주제, 롤플레이 시나리오, 피드백 리포트 자동 생성 |
| 기존 데이터를 활용한 신규 콘텐츠 생성 | 사용자 대화 기록과 학습 이력을 기반으로 맞춤 연습 주제 생성 |
| LLM API 연동 및 프롬프트 설계 | OpenAI Chat/Realtime API 호출, 에이전트별 시스템 프롬프트 관리 |
| 과목별 문항 데이터 구조화 | 영어 학습 단위별 데이터 구조화: 주제, 레벨, 표현, 오류 유형, 피드백 |
| REST API 및 외부 API 연동 | Spring Boot REST API, OpenAI API, Kakao 알림 연동 |
| 교육/에듀테크 서비스 경험 | 영어 스피킹 학습 흐름, 진단, 반복 연습, 성취 지표 설계 |

## 4. 사용자 가치

- 사용자는 별도 앱 설치 없이 웹에서 영어 스피킹 연습을 시작할 수 있다.
- 사용자는 본인의 OpenAI API Key를 등록해 서비스 사용 비용과 권한을 직접 통제할 수 있다.
- 사용자는 대화 종료 후 발음, 표현, 문법, 어휘, 자연스러움에 대한 피드백을 받을 수 있다.
- 관리자는 사용자, 권한, 에이전트, 학습 콘텐츠 구성을 운영 화면에서 관리할 수 있다.

## 5. MVP 범위

### 5.1 사용자 기능

- 회원가입, 로그인, 프로필 관리
- 사용자별 OpenAI API Key 등록 및 암호화 저장
- 영어 학습 에이전트 선택
- 텍스트 기반 AI 채팅
- Realtime API 기반 음성 대화 세션
- 대화 기록 및 transcript 저장
- 대화 종료 후 AI 피드백 요약

### 5.2 관리자 기능

- 사용자 관리
- 역할 및 권한 관리
- 학습 에이전트 기본 프롬프트 관리
- 서비스 설정 관리
- 학습 콘텐츠 또는 대화 주제 관리

### 5.3 AI 기능

- 에이전트별 시스템 프롬프트 적용
- 사용자 레벨과 목표에 맞춘 응답 스타일 조정
- 대화 transcript 기반 피드백 생성
- 자주 틀리는 표현, 추천 표현, 다음 연습 주제 생성

## 6. 도메인 모델 목표

| 도메인 | 설명 |
| --- | --- |
| `credential` | 사용자별 OpenAI API Key 암호화 저장 및 조회 |
| `agent_profile` | 에이전트 이름, 역할, 시스템 프롬프트, 음성, 모델 설정 |
| `conversation` | 대화 세션, 메시지, transcript, 상태 관리 |
| `feedback` | 문법, 어휘, 표현, 발화 흐름, 추천 학습 항목 |
| `realtime` | OpenAI Realtime client secret 발급 및 세션 오케스트레이션 |
| `notification` | Kakao 등 외부 알림 채널 연동 |

## 7. 데이터 구조화 목표

영어 학습 데이터는 단순 대화 로그가 아니라 재사용 가능한 학습 자산으로 저장한다.

| 데이터 | 구조화 기준 |
| --- | --- |
| 대화 주제 | 일상, 비즈니스, 여행, 면접, 발표, 시험 대비 |
| 난이도 | beginner, intermediate, advanced |
| 오류 유형 | grammar, vocabulary, pronunciation, fluency, naturalness |
| 추천 표현 | 원문, 교정문, 대체 표현, 사용 맥락 |
| 학습 이력 | 대화 시간, 완료 여부, 피드백 점수, 다음 추천 주제 |

## 8. 기술 목표

- Backend: Spring Boot 기반 REST API와 AI 연동 모듈을 명확히 분리한다.
- Frontend: Next.js 기반으로 로그인, 관리자, 학습 화면을 일관된 UI로 제공한다.
- Database: PostgreSQL에 사용자, 권한, API Key, 대화, 피드백 데이터를 저장한다.
- Security: 사용자 API Key는 서버에서 암호화 저장하고, 클라이언트에 원문을 다시 노출하지 않는다.
- Deployment: 기존 BeautyBook 인프라를 EnglishAgentHub 운영 인프라로 전환해 배포한다.

## 9. 구현 단계

### Phase 1. AI 대화 기반 완성

- 사용자별 API Key 등록 및 암호화 저장
- OpenAI Chat API 기반 텍스트 대화 안정화
- 기본 영어 학습 에이전트 구성
- 대화 기록 저장

### Phase 2. Realtime 스피킹 코칭

- OpenAI Realtime client secret 발급
- 브라우저 WebRTC 음성 대화 UI
- transcript 저장
- 세션 종료 후 피드백 생성

### Phase 3. 학습 데이터 구조화

- 오류 유형별 피드백 스키마 설계
- 추천 표현과 다음 학습 주제 자동 생성
- 사용자 학습 이력 대시보드
- 에이전트별 프롬프트 관리 화면

### Phase 4. 운영 기능 확장

- Kakao 알림 연동
- 관리자용 학습 콘텐츠 관리
- 운영 로그와 장애 대응 문서 정리
- 배포 자동화 및 운영 체크리스트 개선

## 10. 성공 기준

- 사용자가 로그인 후 본인 API Key를 등록하고 영어 대화를 시작할 수 있다.
- 음성 대화 또는 텍스트 대화가 transcript로 저장된다.
- 대화 종료 후 AI가 학습 피드백과 다음 연습 주제를 생성한다.
- 관리자는 사용자, 권한, 에이전트 설정을 운영 화면에서 관리할 수 있다.
- 운영 환경에서 프론트엔드, 백엔드, DB가 배포 문서 기준으로 안정적으로 동작한다.

## 11. 포트폴리오 관점의 강조 포인트

- 생성형 AI API를 단순 호출하는 수준을 넘어, 프롬프트와 데이터 구조를 서비스 도메인에 맞게 설계한다.
- 사용자별 API Key 암호화, RBAC, 관리자 기능을 포함해 실제 운영 가능한 구조를 갖춘다.
- Realtime API, REST API, PostgreSQL, Next.js UI를 연결한 풀스택 AI 서비스 구현 경험을 보여준다.
- 교육/에듀테크 도메인에서 학습 데이터, 피드백, 개인화 추천 흐름을 설계한 프로젝트로 설명할 수 있다.
