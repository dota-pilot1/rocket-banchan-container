# 메뉴 추가 방법

이 프로젝트의 상단 메뉴는 백엔드 DB의 `menus` 테이블을 기준으로 내려오고, 프론트 헤더가 `/api/menus` 결과를 렌더링한다.

관리자 메뉴를 하나 추가할 때는 보통 아래 순서로 작업한다.

## 1. 백엔드 메뉴 시더 추가

파일:

```text
english-agent-hub-server/src/main/java/com/cj/englishagenthub/config/MenuSeeder.java
```

`defs` 목록에 메뉴를 추가한다.

예시:

```java
new MenuDef(
        "ADMIN_QUESTION_BANK",
        "ADMIN",
        "문제 은행",
        "nav.questionBank",
        "/question-bank",
        "BookOpenCheck",
        RoleSeeder.ROLE_ADMIN,
        2
)
```

필드 의미:

```text
code          메뉴 고유 코드. DB unique 값
parentCode    부모 메뉴 코드. 관리 하위 메뉴면 ADMIN
label         DB 기본 표시명
labelKey      프론트 i18n 키
path          이동 경로
icon          아이콘 이름 또는 관리용 메타 참고값
requiredRole  노출에 필요한 롤
displayOrder  같은 부모 안에서 정렬 순서
```

주의:

- `MenuSeeder`는 `existsByCode`로 이미 존재하는 메뉴는 건너뛴다.
- 이미 DB에 생성된 메뉴의 이름, 경로, 순서를 바꾸려면 관리자 화면의 `메뉴 관리`에서 수정하거나 DB 값을 직접 갱신해야 한다.
- 새 메뉴만 추가하는 경우 서버 재시작만으로 seed 된다.

## 2. 프론트 번역 키 추가

파일:

```text
english-agent-hub-front/src/shared/i18n/resources/ko/nav.ts
english-agent-hub-front/src/shared/i18n/resources/en/nav.ts
english-agent-hub-front/src/shared/i18n/resources/ja/nav.ts
english-agent-hub-front/src/shared/i18n/resources/zh/nav.ts
```

`labelKey`에 맞는 키를 추가한다.

예시:

```ts
questionBank: "문제 은행",
```

백엔드 `labelKey`가 `nav.questionBank`라면 `nav.ts` 안에는 `questionBank` 키가 있어야 한다.

## 3. 헤더 관리 메뉴 메타 추가

파일:

```text
english-agent-hub-front/src/widgets/header/ui/Header.tsx
```

관리 드롭다운 카드에 설명과 아이콘을 보여주려면 `adminMenuMeta`에 추가한다.

예시:

```ts
import { BookOpenCheck } from "lucide-react";

const adminMenuMeta = {
  ADMIN_QUESTION_BANK: {
    description: "문제 저장, 조회, 유사문항 PoC 데이터를 관리합니다.",
    icon: BookOpenCheck,
  },
};
```

메타를 추가하지 않아도 메뉴 이동은 가능하지만, 관리 메가메뉴에서 기본 설명과 기본 아이콘으로 표시된다.

## 4. 페이지 라우트 생성

Next.js App Router 기준으로 메뉴 경로와 같은 폴더를 만든다.

예시:

```text
english-agent-hub-front/src/app/question-bank/page.tsx
```

관리자 전용 페이지라면 `RequireRole`로 감싼다.

```tsx
import { RequireRole } from "@/widgets/guards/RequireRole";

export default function QuestionBankPage() {
  return (
    <RequireRole roles={["ROLE_ADMIN"]}>
      <main>문제 은행</main>
    </RequireRole>
  );
}
```

## 5. API가 필요한 메뉴라면 프론트 API 클라이언트 추가

엔티티별 API 파일을 만든다.

예시:

```text
english-agent-hub-front/src/entities/question/api/questionApi.ts
```

공통 axios 인스턴스는 아래를 사용한다.

```ts
import { api } from "@/shared/api/axios";
```

## 6. 백엔드 API가 필요한 메뉴라면 컨트롤러 추가

관리자 전용 API는 컨트롤러에 권한을 명시한다.

```java
@RestController
@RequestMapping("/api/questions")
@PreAuthorize("hasRole('ADMIN')")
public class QuestionController {
}
```

권한 체크가 동작하려면 `SecurityConfig`에 `@EnableMethodSecurity`가 있어야 한다. 현재 프로젝트에는 이미 설정되어 있다.

## 7. 확인 순서

서버/프론트 변경 후 아래를 확인한다.

```bash
cd english-agent-hub-server
./gradlew compileJava
```

```bash
cd english-agent-hub-front
npm run lint
npx tsc --noEmit
```

그 다음 서버를 재시작한다.

```text
관리자 계정 로그인
→ 상단 관리 메뉴 열기
→ 새 메뉴 노출 확인
→ 메뉴 클릭
→ 페이지 접근 확인
```

일반 사용자 계정으로 접근하면 관리자 페이지는 `/unauthorized`로 이동하는 것이 정상이다.
