## 인계 vs 신규 (한 줄 요약)

- **인계**: 도메인, Route 53, ACM, CloudFront 배포, S3 버킷, EC2, Nginx — 그대로 사용.
- **신규**: PostgreSQL 컨테이너(5436), 백엔드 systemd(4301), `.env` — EC2 안에서만 추가.
- **중단**: BeautyBook 백엔드 systemd, Nginx의 BeautyBook 라우팅 — 보존하되 OFF.

## 트래픽 경로

1. 사용자가 `https://dxline-tallent.com/...` 접속.
2. Route 53이 CloudFront로 라우팅.
3. CloudFront의 behavior:
   - `/api/*` → EC2 origin (Nginx) → `127.0.0.1:4301` (Spring Boot)
   - 그 외 → S3 origin (`beauty-book-hair-front`) → Next.js static export
4. 백엔드는 EC2 내부 Postgres `:5436`와만 통신.
5. OpenAI Realtime은 클라이언트가 직접 OpenAI에 WebRTC 연결 (Nginx 경유 안 함).

## 보안

- ACM 인증서로 CloudFront ↔ 사용자 구간 HTTPS.
- Nginx ↔ Spring Boot는 EC2 내부 통신 (HTTP 무방).
- Spring Boot ↔ Postgres는 EC2 내부 통신.
- 사용자 OpenAI API 키는 `App.security.api-key-secret`으로 AES-GCM 암호화되어 DB에 저장.
- JWT는 `JWT_SECRET`으로 서명.

## 환경 변수 흐름

```text
.env (EC2)
  ├─ OPENAI_API_KEY        ← 시스템 폴백 키
  ├─ API_KEY_SECRET        ← 사용자 키 암호화 시크릿
  ├─ JWT_SECRET            ← 토큰 서명
  ├─ ALLOWED_ORIGIN        ← https://dxline-tallent.com
  └─ (DB는 application.yaml 기본값 사용 — localhost:5436)
```

```text
프론트 빌드 시 (로컬)
  └─ NEXT_PUBLIC_API_URL=https://dxline-tallent.com
```

## EC2 메모리 운영

- EC2가 t3.small(2GB)이므로 Java 두 개를 동시에 띄우지 않는다.
- BeautyBook(`beautybook.service`)는 `stop` + `disable`로 꺼둔다.
- 필요 시 swap 1~2GB를 활성화한다.

## 운영 중단(BeautyBook) 정책

- BeautyBook 데이터는 보존한다.
- `beauty-book-postgres` 컨테이너는 그대로 둔다 (자동 재시작 정책에 따라 살아있을 수 있음).
- 컨테이너를 꺼두고 싶으면 `docker stop beauty-book-postgres` (제거 X).
- 백업 또는 마이그레이션이 필요할 때 다시 켜서 데이터를 꺼낼 수 있다.
