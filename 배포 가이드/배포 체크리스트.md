# 배포 체크리스트 (BeautyBook 인계)

전체 흐름: **EC2 셋업 → 백엔드 → 프론트 → 라우팅 전환**.

## 0. 사전 확인 (로컬)

- [ ] 백엔드 컴파일 통과: `cd english-agent-hub-server && ./gradlew compileJava`
- [ ] 프론트 타입체크 통과: `cd english-agent-hub-front && npx tsc --noEmit`
- [ ] 운영 시크릿 값 준비
  - `OPENAI_API_KEY` (시스템 폴백용)
  - `JWT_SECRET` (32바이트 이상 랜덤)
  - `API_KEY_SECRET` (32바이트 이상 랜덤, 한 번 정하면 바꾸기 어려움 — 저장된 유저 키 복호화 불가)

## 1. EC2 — EnglishAgentHub 인프라 추가

SSH 접속: `ssh -i "배포 가이드/hibot-d-server-key.pem" ubuntu@13.209.195.64`

- [ ] Postgres 컨테이너 추가 (`english-agent-hub-postgres`, 5436)
- [ ] `/home/ubuntu/english-agent-hub/` 디렉터리 생성
- [ ] `/home/ubuntu/english-agent-hub/.env` 작성 (`.env.prod.example` 참조)
- [ ] `englishagenthub.service` systemd 파일 작성 (백엔드 배포.md 참조)
- [ ] `sudo systemctl daemon-reload`

> BeautyBook DB 컨테이너는 **건드리지 않는다**. 그대로 둔다.

## 2. 백엔드 빌드 + 업로드

- [ ] 로컬: `cd english-agent-hub-server && ./gradlew build -x test`
- [ ] `build/libs/english-agent-hub-server-*.jar` → `scp`로 EC2의 `/home/ubuntu/english-agent-hub/app.jar`
- [ ] EC2에서 `sudo systemctl start englishagenthub`
- [ ] EC2에서 `sudo systemctl enable englishagenthub`
- [ ] EC2에서 검증: `ss -tlnp | grep ':4301'` → LISTEN 확인
- [ ] EC2에서 검증: `curl -sf http://127.0.0.1:4301/swagger-ui/index.html | head -1`

## 3. 프론트 빌드 + S3 sync

- [ ] 로컬: `cd english-agent-hub-front`
- [ ] `export NEXT_PUBLIC_API_URL=https://dxline-tallent.com`
- [ ] `npm run build` → `out/` 생성 확인
- [ ] **BeautyBook S3 비우기 (선택)**: `aws s3 rm s3://beauty-book-hair-front/ --recursive`
  - 또는 `--delete` 옵션으로 sync하면 같은 효과
- [ ] `aws s3 sync out/ s3://beauty-book-hair-front/ --delete`
- [ ] `aws cloudfront create-invalidation --distribution-id E11NF3HMOB52NI --paths "/*"`

## 4. BeautyBook → EnglishAgentHub 라우팅 전환

EC2에서:

- [ ] BeautyBook 백엔드 중지 + 비활성화 (데이터·컨테이너 보존)
  ```bash
  sudo systemctl stop beautybook
  sudo systemctl disable beautybook
  ```
- [ ] Nginx `dxline-tallent.com` server 블록의 `/api/` → `127.0.0.1:4301`로 변경 (Nginx 설정.md 참조)
- [ ] `sudo nginx -t && sudo systemctl reload nginx`

## 5. 검증

- [ ] `https://dxline-tallent.com` 접속 → EnglishAgentHub 대시보드 로딩
- [ ] `https://dxline-tallent.com/api/site-settings` 응답 확인
- [ ] 로그인/회원가입 동작
- [ ] AI 채팅(`/api/ai/chat/stream`) 동작
- [ ] 프로필 → API 키 저장 → `유효성 확인` 통과
- [ ] 캐릭터 추가/수정/삭제 동작
- [ ] Realtime client secret 발급 (`/api/realtime/client-secret`) 동작

## 6. 운영 주의사항

- [ ] EC2 메모리 확인 (Java 두 개 동시 운영은 안 함 — BeautyBook 중지 상태)
- [ ] swap 설정 (없으면 추가 권장)
- [ ] CloudFront 캐시 무효화는 매 배포마다 필수 (정적 파일 갱신 반영)
- [ ] `API_KEY_SECRET`은 절대 바꾸지 않는다 (저장된 사용자 OpenAI 키 복호화 불가)
- [ ] BeautyBook 데이터 정리는 별도 작업으로 분리 (지금은 보존)
