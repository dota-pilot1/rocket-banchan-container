# 백엔드 배포 가이드

## 사전 조건

- BeautyBook EC2(`13.209.195.64`)에 SSH 접근 가능 (PEM 키)
- 로컬에 JDK 21 + Gradle Wrapper 동작
- 운영용 `OPENAI_API_KEY`, `JWT_SECRET`, `API_KEY_SECRET` 준비

## EC2 작업 (최초 1회)

### 1) PostgreSQL 컨테이너

```bash
ssh -i "배포 가이드/hibot-d-server-key.pem" ubuntu@13.209.195.64

# 컨테이너 확인 (BeautyBook용은 이미 있을 것)
docker ps -a | grep postgres

# EnglishAgentHub용 신규 추가
docker run -d \
  --name english-agent-hub-postgres \
  --restart unless-stopped \
  -e POSTGRES_DB=english_agent_hub \
  -e POSTGRES_USER=postgres \
  -e POSTGRES_PASSWORD=postgres \
  -p 5436:5432 \
  -v english-agent-hub-pgdata:/var/lib/postgresql/data \
  postgres:16

# 동작 확인
docker exec english-agent-hub-postgres psql -U postgres -d english_agent_hub -c '\dt'
```

### 2) 디렉터리 + env 작성

```bash
sudo mkdir -p /home/ubuntu/english-agent-hub
sudo chown ubuntu:ubuntu /home/ubuntu/english-agent-hub
cd /home/ubuntu/english-agent-hub

# .env 작성 (배포 가이드/.env.prod.example 참고)
nano .env
chmod 600 .env
```

`.env` 최소 항목:

```env
JWT_SECRET=...32바이트 이상...
API_KEY_SECRET=...32바이트 이상... (한 번 정하면 변경 금지)
OPENAI_API_KEY=sk-...
ALLOWED_ORIGIN=https://dxline-tallent.com
SERVER_PORT=4301

# 선택
OPENAI_CHAT_MODEL=gpt-5-mini
OPENAI_TRANSLATION_MODEL=gpt-5-nano
OPENAI_TRANSCRIBE_MODEL=gpt-4o-mini-transcribe
OPENAI_REALTIME_MODEL=gpt-realtime-2
OPENAI_REALTIME_VOICE=marin
```

### 3) systemd 서비스 작성

```bash
sudo nano /etc/systemd/system/englishagenthub.service
```

```ini
[Unit]
Description=EnglishAgentHub Spring Boot
After=network.target docker.service
Requires=docker.service

[Service]
Type=simple
User=ubuntu
WorkingDirectory=/home/ubuntu/english-agent-hub
EnvironmentFile=/home/ubuntu/english-agent-hub/.env
ExecStart=/usr/bin/java -Xmx512m -Xms256m -jar /home/ubuntu/english-agent-hub/app.jar
Restart=on-failure
RestartSec=10
StandardOutput=append:/home/ubuntu/english-agent-hub/app.log
StandardError=append:/home/ubuntu/english-agent-hub/app.log

[Install]
WantedBy=multi-user.target
```

```bash
sudo systemctl daemon-reload
```

## 로컬 빌드 + 업로드

```bash
cd english-agent-hub-server
./gradlew clean build -x test
ls -lh build/libs/   # english-agent-hub-server-*.jar 확인

# EC2로 업로드
scp -i "../배포 가이드/hibot-d-server-key.pem" \
  build/libs/english-agent-hub-server-*.jar \
  ubuntu@13.209.195.64:/home/ubuntu/english-agent-hub/app.jar.new

# EC2에서 swap (무중단)
ssh -i "../배포 가이드/hibot-d-server-key.pem" ubuntu@13.209.195.64 << 'EOF'
cd /home/ubuntu/english-agent-hub
mv app.jar.new app.jar
sudo systemctl restart englishagenthub
sleep 5
sudo systemctl status englishagenthub --no-pager | head -20
ss -tlnp | grep ':4301'
EOF
```

## 시작 / 활성화

```bash

sudo systemctl start englishagenthub
sudo systemctl enable englishagenthub

# 이후 재배포는 restart
sudo systemctl restart englishagenthub
```

## DB 초기화

`spring.jpa.hibernate.ddl-auto=update`이므로 첫 기동 시 모든 테이블이 자동 생성된다.

생성되는 주요 테이블:
- `users`, `roles`, `permissions`, `role_permissions`
- `characters`, `character_skills`, `character_starter_prompts`
- `menus`, `menu_permissions`
- `site_settings`

`users.openai_api_key` 컬럼은 `User` 엔티티에 정의돼 있어 자동 마이그레이션된다.

## 운영 시 흔한 문제

- **`OPENAI_NOT_CONFIGURED` (503)**: 사용자가 개인 키도 안 넣었고 `.env`에도 `OPENAI_API_KEY`가 비어 있다. `.env`에 시스템 키를 채우거나, UI에서 사용자가 개인 키 등록.
- **`AI_REQUEST_FAILED` (502)**: OpenAI 호출 실패. 모델 이름·키 권한 확인. `app.log`에 원본 응답이 찍힌다.
- **DB 연결 실패**: `english-agent-hub-postgres` 컨테이너가 안 떠 있다. `docker start english-agent-hub-postgres`.
- **메모리 부족(OOM)**: EC2 2GB. BeautyBook이 켜져 있는지 확인. swap 추가.
